Strategic Priority
Design Principles
① Server-side attribution backup — Cookies break (ITP, ad blockers, cross-domain). Attribution survives by storing click IDs server-side too. Cookies are primary; the server-side store is the safety net.
② Nothing critical lives only in Marketo — Attribution, enrichment, routing, and Lead Source mapping work regardless of which MAP processes the lead. Marketo is right today. The system doesn't collapse if that changes.

Cross-Domain Handoff Pattern
Any link crossing domains appends ?awx_vid={visitorID}. Destination page reads param, re-sets first-party cookie on new domain. All domains resolve attribution through the central API using the visitor ID. One GTM tag handles link decoration automatically.

⚠ Critical Sync Direction Rule — Most Likely Root Cause
ALL attribution fields sync Marketo → SFDC only. Block SFDC → Marketo sync on these fields. Without this, SFDC Workflows, Process Builder, or Apex triggers can overwrite attribution data back into Marketo on the next sync cycle — even if Marketo-side bidirectional sync is unchecked. This is the most likely root cause of the blank Lead Source fields Wei Kuan described.

Smart Campaign: Lead Processing Flow

TRIGGER: Fills Out Form (any form in global set) Step 1: Webhook → Lead Source mapping API Send: utm_source, utm_medium, utm_campaign Receive: Lead_Source, Lead_Source_Segment Write to lead record immediately Step 2: Stamp First Touch IF FT_Source IS EMPTY: Set FT_Source = UTM_Source Set FT_GCLID = GCLID Set FT_Date = {{system.dateTime}} Set FT_Lead_Source = Lead_Source Step 3: Stamp Last Touch (always) Set LT_Source = UTM_Source Set LT_GCLID = GCLID Set LT_Lead_Source = Lead_Source Step 4: Spam Check IF Spam_Score ≥ 40: Set Is_Spam = TRUE Add to Spam Quarantine END (do not sync to SFDC)
Step 5: Enrichment Webhook → Clearbit Send: email, company name Receive: size, revenue, industry, HQ, tech stack Timeout: 5s → skip, queue for batch Never block scoring on enrichment failure Step 6: Lead Scoring Behavioral + firmographic dimension Enterprise signals → higher score Step 7: Route + Sync IF Company_Size > 1000 OR Revenue > $100M: → Enterprise AE team ELSE IF Company_Size > 50: → Mid-market team ELSE: → SMB / self-serve nurture Sync Lead to Salesforce Step 8: LinkedIn Lead Gen (separate trigger) Source = LinkedIn LaunchPoint connector Verify li_fat_id field mapping in LaunchPoint Fallback: hashed email for CAPI matching Daily reconciliation: LinkedIn count vs Marketo

Lead Enrichment — Enabling the Upmarket Push

Without enrichment data, you can't route enterprise leads differently, score them properly, or give sales context before the first call. Airwallex is going upmarket — enrichment is now table stakes.

🏢

Company Data

Size, revenue, industry, HQ location, tech stack, funding stage. Source: Clearbit Enrichment API (real-time webhook, sub-second).

👤

Person Data

Title normalization → seniority level, LinkedIn URL, direct dial. Source: Clearbit for real-time; ZoomInfo for batch and phone data.

🎯

Intent Data (Future)

Account-level intent signals (6sense or Bombora). Know which companies are researching "cross-border payments" right now. Layer into lead scoring.

reCAPTCHA caveat

Part 8

Debugging: Google Ads ↔ SFDC Mismatch

Scenario: A campaign is generating leads, but Google Ads conversion data doesn't match Salesforce opportunities. Step-by-step diagnostic — grounded in Airwallex's specific known failure points.

Layer 1
Google Ads Config
Auto-tagging enabled? Conversion action type (tag-based vs. offline import)? If tag-based: GTM tag firing on confirmation page? If offline import: import job running? Conversion window set correctly for 60+ day enterprise cycles?
Layer 2
Landing Page
Submit a test lead with UTM params + GCLID in URL. Check browser dev tools → Form POST data. Is GCLID being captured in the hidden field? Watch for URL fragments (#), double-encoded params, URL shorteners stripping the GCLID.
Layer 3
Marketo Activity Log
"Filled Out Form" — are GCLID + UTM fields populated? "Change Data Value" — did any subsequent campaign overwrite GCLID or Lead Source? "Sync Lead to SFDC" — sync successful? Pull 10 leads from the campaign — what % have GCLID populated? If <80%, capture is broken.
Layer 4 ← Most Likely Here
Marketo → SFDC Sync
Check Marketo sync error log. Verify field mapping: Marketo GCLID → SFDC GCLID__c. Audit every Workflow, Process Builder, and Apex trigger that fires on lead creation and may overwrite Lead Source, Lead Source Segment, or GCLID. This is the most likely root cause given Wei Kuan's description.
Layer 5
Salesforce
Lead Source populated? If blank: enable Field History Tracking — it shows exactly when and by what process the field was cleared. Regional routing check: is Canada's pipeline showing under US because the lead was routed to a US rep? Preserve Original_Region__c separately from Routed_Region__c.
Layer 6
Offline Conversion Import
Is import configured? Import log showing errors? GCLID matching fails silently if GCLID is blank in SFDC. Note: Legacy manual offline conversion import was sunset May 2025. If Airwallex is still on legacy, migration to Enhanced Conversions is required.

Part 9 — 2–3 min max

Without Marketo — The Composable Future

The architecture in Parts 1–8 already makes Marketo a swappable processing node. Attribution lives server-side. Lead Source mapping is API-driven. Enrichment is webhook-based. If Marketo stays, it processes and syncs. If it goes, nothing upstream breaks.

My Recommendation
Don't propose ripping Marketo out today. Instead, show the architecture is already designed for swappability. The decision to swap can be made later based on cost, scale, and capability needs — without re-architecting the system.

Honest Assessment

Risks & Assumptions

Being honest about what could go wrong builds trust faster than pretending everything works perfectly.

Assumptions I'm Making

✓ SFDC admin access allows implementing field-level security on attribution fields

✓ LinkedIn LaunchPoint issues are fixable at the connector/config level — not a fundamental integration failure

✓ Sales team accepts locked-down attribution fields (requires Sales Ops governance buy-in)

✓ Airwallex can procure an enrichment vendor (Clearbit, ZoomInfo, or similar)

Risks to Flag

Historical data: If millions of leads have incorrect Lead Source from the manual upload era, reporting won't be trustworthy until a cleanup pass is complete. Potentially weeks of effort.

200+ UTM mappings: If taxonomy is large, webhook lookup is better than Choice steps — but adds an API dependency that needs monitoring.

CN timeline: PIPL + Baidu + Alibaba Cloud may require legal review and local engineering support. Not committing to a CN timeline in 90 days without more info.

Server-side API: Medium-term investment, not a Day 1 build. Quick wins come from fixing Marketo/SFDC config first.

What I'd do if any assumption is wrong
Every assumption is something I'd verify in the Day 1 audit. If something doesn't hold, I adjust scope in Week 2 rather than push forward with a plan that doesn't fit reality. I've built similar systems at LastPass (20M+ record sync stabilization) and across 20+ enterprise clients at Inverta — the diagnostic process is always the same. The findings differ.
